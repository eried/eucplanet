EUC Planet HUD

Companion app for Motoeye E6 and compatible Android-based head-up displays.

All releases on GitHub

Which file do I download?

Each EUC Planet release ships two APKs side by side:

  • EUC-Planet-phone-motoeye-preview-N.apk — install on the rider's phone.
  • EUC-Planet-hud-motoeye-preview-N.apk — install on the Motoeye HUD device.

The numbers must match: a phone APK from preview-2 needs the HUD APK from preview-2 (or newer in the same major version line). The phone will tell you if the HUD is too old.

Always pick the latest release at the top of the page. Older releases are kept around for archaeology; install the most recent one unless you have a specific reason not to.

Install on the Motoeye E6

  1. Plug the Motoeye into your computer with a USB-C cable. Power it on. After a moment it appears as a drive (sometimes labelled "Motoeye E6").
  2. Copy EUC-Planet-hud-motoeye-preview-N.apk into the root of that drive.
  3. Unplug. On the Motoeye, go to Settings → Application Management → App Installer, then select the APK file and confirm.
  4. The EUC Planet icon will appear in the launcher.

Install on the phone

  1. Open EUC-Planet-phone-motoeye-preview-N.apk on the phone. Android may need permission to install from this source the first time — grant it.
  2. If you have a previous EUC Planet install from a different source, you may need to uninstall it first when signatures don't match.
  3. Open the app once after install so the foreground service initialises.

Pair the two

  1. Turn on the phone's WiFi hotspot.
  2. Connect the Motoeye to that hotspot.
  3. Launch EUC Planet on the HUD. The disconnect splash shows the HUD's IP (something like 192.168.43.x:28080).
  4. On the phone, open EUC Planet → Settings → Integration → Motoeye HUD. Type the HUD's IP into Device IP, leave the Port at 28080.
  5. Toggle Enable data link on. Telemetry starts flowing within a couple of seconds.

Keeping the apps in sync

The phone app and the HUD app talk a versioned protocol. When the link is up, each side checks the other's version and decides:

  • Same version: link works, nothing to do.
  • Different minor (e.g. phone 1.2 ↔ HUD 1.1): link still works, the older side just misses any new features. The app shows a soft "update available" hint.
  • Different major (e.g. phone 2.x ↔ HUD 1.x): the older side blocks rendering live data and tells you to update. Pick the matching APKs from the latest release on GitHub.

Easiest rule: when you update one APK, install the matching APK from the same release on the other side.

Protocol reference

If you want to build a third-party HUD or a phone client that speaks the same wire format, the reference lives in the source tree:

  • Protocol specification — transport, frame format, command set, embedded overlay format, versioning rules.
  • Shared :hud-protocol module — Kotlin source for both sides of the link.
