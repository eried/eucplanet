Service Mode

EUC Planet · diagnostics and protocol probing

Sends real bytes to your wheel. You can change settings unexpectedly. Use it if you know what you are doing.

Hidden dialog that captures every BLE byte to/from the wheel, lets you fire test commands by hand, and exports a .txt log. For bug reports and protocol research.

Enter / exit

Dashboard version number → About → tap-and-hold the logo (drag off to abort). Confirm the warning. Version pulses red. Tap it again any time to reopen. Stop icon exits and clears the log.

The log keeps capturing in the background while you use the app normally.

What gets logged

  • RX / TX raw bytes (one BLE chunk per line)
  • CMD test command + label
  • NOTE reassembled frames + internal events
  • COMMENT your free-text notes (or attached snapshots — see below)
  • INFO session metadata: phone, Android, RAM/storage, BLE adapter, paired Wear node, wheel firmware

Nothing leaves the device until you tap Share.

Commands tab

Per-wheel-family preset list. Pick a family in the dropdown (single-family adapters show it disabled), fire a command, watch the log, comment what changed physically. The dashboard's Light/Horn/Lock keep working in parallel.

Inspect tab

Live byte-grid of the incoming stream. Pick family + message type. Tap a byte cell to log byte N = X. The min/max range filter hides cells outside the range and shows a hidden-count badge. Use it to find which offset carries which value.

Raw tab

Type hex, send it. 60 50 01 01, 6050-0101, 60500101 all parse to the same four bytes. 64-byte cap. The read-only Bytes-to-send line shows the final on-wire frame after wrapping.

Wrap mode

  • Literal — exactly as typed (full frame, your own checksum).
  • V2 extended — InMotion V12 / V14 / P6: prepends aa aa 16 LL 02 21, appends XOR checksum.
  • V14 short — legacy InMotion control flags.
  • KingSong / Ninebot legacy / InMotion V1 — their respective framing.

Example. V2 extended around 60 50 01 01 (P6 headlight on):

aa aa 16 06 02 21 60 50 01 01 [xor]

Presets

Common V2-ext sub-commands available as chips:

  • 60 50 00 00 / 60 50 01 01 — headlight off / on
  • 60 51 18 01 — horn beep
  • 60 2f 00 / 01 — auto-headlight off / on
  • 60 24 00 / 01 — 25 km/h clamp off / on
  • 02 06 — info bundle
  • 02 07 — realtime telemetry
  • 20 20 — settings page A

Attach data to a comment

The comment row has an Attach picker. Drops a structured snapshot into the log as a COMMENT:

  • App config
  • Wheel state
  • Trips summary
  • Installed apps
  • Crashes

The Wear OS companion also announces its build info on each launch as a NOTE entry, so the watch firmware shows up alongside the phone's INFO line.

Sharing

Share icon in the header → .txt via the Android share sheet.

For developers

Per-wheel presets live in WheelAdapter.getDiagnosticCommands(). Inspect message types come from WheelAdapter.inspectMessageTypes(). Add a wrap mode in WheelDiagnosticsViewModel.WrapMode + wrap().
